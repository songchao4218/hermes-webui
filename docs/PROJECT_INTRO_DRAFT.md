## 为什么做这个项目？

市面上不缺 AI 助理，但真正能**安静地、稳定地、在本地跑起来干活**的，几乎没有。

在使用 OpenClaw 的过程中，我们（作为真实用户）遇到了以下无法忍受的问题：

### 1. 不知道它在干什么

OpenClaw 经常卡死——不报错，不提示，没有任何进度反馈。关掉怕打断任务，等下去又不知道要等多久。这种"黑盒"体验，对于一个每天要用的工具来说，是致命的。

### 2. WebUI 配置像在读论文

OpenClaw 的 Web 界面试图把所有设置都塞进去，结果是：新用户根本不知道从哪里开始，老用户也不想每次都翻半天。一个本地助理的配置应该是**开箱即用**的，而不是要求用户先成为专家。

### 3. 在线 API 模式不适合所有人

大多数 AI 助理默认连接云端 API——这意味着持续的费用、对网络稳定性的依赖，以及数据离开本地的隐私风险。对于中国大陆用户来说，海外 API 的网络问题更是家常便饭，成天光折腾网络和模型了，根本没法安心干活。

### 4. 技能生态的"丰富"反而是负担

OpenClaw 的 Skill 数量很多，但找到一个真正合适的却很费力——需要逐个试用、检查安全性、排查兼容性。对普通用户来说，这不是"生态丰富"，而是"选择疲劳"。

## 马鞍 的设计哲学

基于以上痛点，马鞍 选择了一条不同的路：**不做最全的，做最稳的。**

### 原则一：透明可控，拒绝黑盒

Hermes Agent 本身就以稳定和快速响应著称。在此基础上，马鞍 增加了：

- **实时进度指示** — 你随时知道 Agent 在做什么（思考中、调用技能中、等待模型响应...）
- **延迟和资源监控** — 底部状态栏实时显示响应延迟、显存占用、上下文长度
- **预计等待时间**（规划中）— 对于耗时任务，给出预估时间，让你可以安心挂机去睡觉

> 用户体验的核心不是花哨的动画，而是**让人心里有底**。

### 原则二：全本地，零费用，零折腾

马鞍 只专注一件事：**让 Ollama 本地模型跑满你的硬件**。

- 安装时自动检测显存，推荐最适合的 Gemma 模型（4B / 12B / 27B）
- 不需要任何 API Key，不依赖任何云端服务
- 配置极简——既然是本机跑，那就按本机配置拉满，干活为主

Google Gemma 系列模型从小到大几乎全覆盖，免费、开源、多语言支持好，完全够日常使用。

### 原则三：技能靠积累，不靠囤

马鞍 不鼓励一次装一堆技能。正确的方式是：

- 起步时安装 Hermes 官方的几个核心技能（天气、搜索、文件管理）
- 在实际使用中，让 Agent **自动生成**适合你工作流的自定义技能
- 技能越用越多，Agent 越用越懂你——这才是"个人助理"该有的进化方式

### 原则四：记忆是灵魂

Agent 最核心的价值不是模型有多大，而是**它记住了什么**。

马鞍 与 Hermes CLI 共享同一套记忆系统（SOUL.md / MEMORY.md / USER.md），确保：

- 在终端里教会它的事情，Web 里也知道
- 在 Web 里积累的上下文，终端里也能用
- Agent 的人格、知识、用户档案，始终是同一份

**你的 Agent 不会因为换了个界面就失忆。**

### 原则五：好看，但不喧宾夺主

马鞍 的界面采用 Cyber-noir 设计语言（由 Google Stitch 生成），追求：

- **深色护眼**基底，适合长时间使用
- **极简交互**，不堆功能入口
- **细节丰富**但不干扰——扫描线叠加、毛玻璃效果、呼吸灯状态指示
- **支持用户自定义配色**（规划中）——你的助理，你的风格

## 安装理念

我们的安装逻辑是：**先检测，再决定，最后引导。**

1. **检测环境** — 自动检查 Python 版本、显卡驱动、可用显存
2. **智能推荐** — 根据硬件配置推荐最佳模型（而不是让用户自己去查兼容表）
3. **一键部署** — 自动安装 Ollama、拉取模型、配置 Hermes、启动 WebUI
4. **验证确认** — 安装完成后自动运行一次对话测试，确保一切正常

目标是：**从 `git clone` 到和 Agent 对话，不超过 10 分钟。**

## 目标用户

- 想要一个**稳定、好用、不花钱**的本地 AI 助理
- 不想折腾 API Key、网络代理、复杂配置
- 希望 Agent 能**记住自己、越用越聪明**
- 对界面有审美要求，但更在乎实用性
- 开发者、创作者、学生、或任何想让 AI 真正融入日常工作的人
